# SlidePad 칸반 패널 수동 QA 체크리스트 (v2)

## 기본 토글
- [ ] Cmd + Shift + K 한 번에 슬라이드+페이드 인 (0.20초).
- [ ] 다시 누르면 슬라이드+페이드 아웃.
- [ ] View 메뉴 "SlidePad 칸반 토글" 도 동일 동작.

## non-activating
- [ ] Safari 활성 상태에서 호출해도 Safari 활성 유지.
- [ ] 카드 제목 입력 등 텍스트 입력 정상.

## 자동 숨김
- [ ] 패널 바깥 클릭 시 슬라이드 아웃.
- [ ] 패널 안쪽 클릭은 닫히지 않음.
- [ ] Esc 키로 슬라이드 아웃.
- [ ] 카드 편집 시트 열린 상태에서 외부 클릭/Esc 가 패널을 닫지 않음 (시트만 닫힘).
- [ ] 라벨 매니저 열린 상태도 동일.
- [ ] 핀 버튼 켜면 외부 클릭/Esc 무시.
- [ ] 핀 해제 시 자동 숨김 복원.

## 데이터 공유 (Store)
- [ ] 패널에서 카드 추가 → 메인 윈도우 칸반에 즉시 반영.
- [ ] 메인 → 패널 방향 동일.
- [ ] 패널 검색어가 메인 검색에 영향 없음.
- [ ] 패널 편집 시트가 메인 편집 시트와 독립.
- [ ] 패널 드래그 중 메인 드래그 상태 없음.

## 풀스크린 Space
- [ ] Safari 풀스크린 위에서 호출 시 패널 표시 (macOS 13/14/15/26).
- [ ] 닫은 후 Safari 포커스 유지.

## 멀티 디스플레이
- [ ] 마우스 위치 정책 시 마우스 있는 디스플레이에서 슬라이드 인.
- [ ] 메인 디스플레이 정책 시 메인에서.
- [ ] Settings 에서 특정 디스플레이 UUID 고정 시 해당 디스플레이.
- [ ] 디스플레이 연결/해제 시 다음 호출에서 올바른 디스플레이.
- [ ] 표시 중 디스플레이 추가/제거 시 frame 재계산.

## 한글 IME
- [ ] 한글 조합 중 외부 클릭이 입력을 끊지 않음 (조합 종료 후 닫힘).
- [ ] 일반 입력/영문 토글 정상.

## sleep/wake
- [ ] 노트북 뚜껑 닫았다 열기 → 단축키 계속 동작.

## 단축키 충돌
- [ ] 다른 앱이 Cmd+Shift+K 점유 시 Settings 에 등록 실패 경고.
- [ ] View 메뉴 fallback 동작.

## 워밍업
- [ ] 앱 실행 직후 ~1초 후 첫 호출 즉시 슬라이드 인 시작 (지연 없음).

## 빌드/설치
- [ ] make test 모두 통과.
